全体を貫く設計方針が必要

クラスの組み合わせで作っていく方法は、デザインの段階からそれを想定していないと非常に難しい。

うまくいけば非常に技巧的な、記述量の少ないCSSが組めると思うが、それを求められることはほとんどない。

現実のクライアントワークには合わなかった。

BEMとの出会い

BEM概要

MindBEMding

BEMという設計手法を多少柔軟にしたもの。

BEM

クラス名に悩まなくなった 最初にブロック分割を考えることで、全体を見渡しながら作業できるようになった 各ブロックは、ブロック外とカスケーディングすることがなく疎結合→壊れにくい（図:text-redみたいなの）

設計手法は他にもいろいろあります

    ブロック（モジュール/コンポーネントなどと呼ぶことも）に分割して、擬似的に名前空間を作り出す、それらを疎結合にする クラスの命名規則を設定する、 といったあたりはどれも共通です。
